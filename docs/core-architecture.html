核心执行链

  • • `src/main.rs` 解析 `dump/fix/offsets`，把 UID、PID、libart、输出目录和探针模式收敛为 `DumpConfig`。
  • • `src/art.rs` 从 `libart.so` 解析符号、签名或字符串引用，产出 ART hook 目标与 Android 13+ 默认布局。
  • • `src/dump.rs` 通过 Aya 加载 `build.rs` 编译出的 BPF 对象，写入 `config_map` 与 `art_layout_map` 后 attach uprobe。

采集与降级

  • • `bpf/bpf.c` 从 ART 方法、DexFile 生命周期和 libc native buffer 行为中产生 DEX、方法字节码、布局诊断和 native buffer 事件。
  • • Rust 侧持续 drain RingBuf，按 DEX begin 地址去重、拼 chunk、生成 `dex_*.dex` 与 `dex_*_code.json`。
  • • `--probe-mode maps-only` 直接走 `/proc/<pid>/maps` 扫描，不加载 BPF；`lifecycle` 缩小 attach 面。

修复与产物

  • • `src/fix.rs` 匹配 `dex_*.dex` 与方法 JSON，把捕获到的 code item 写回 DEX 并重新计算 SHA1/Adler32。
  • • `src/dex.rs` 提供 DEX header、字符串、类型、方法签名和 class data 解析能力，支撑方法命名与覆盖率统计。
  • • 输出目录包含原始 DEX、`fix/` 中间修复件、`final/` 最终 DEX 和缺失方法覆盖报告。
